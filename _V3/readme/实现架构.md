# V3 实现架构

更新时间：2026-02-28

## 1. 总览

V3 拆成两个并列工程：

```text
_V3/   -> 检索、训练、评测、导出、服务
_chat/ -> 本地聊天 UI、DeepSeek API Agent、会话保存
```

其中：

- `_V3` 负责“把 memory 组织好、检索好、排序好、导出好”
- `_chat` 负责“把 `_V3` 当作记忆找回器，用在聊天界面里”

## 3. V3 主链

### 3.1 数据生产

`build_dataset.py`

输入：
- 顶层 `data/` 中共享的原始或已处理数据

输出：
- `data/V3/processed/memory.jsonl`
- `data/V3/processed/query.jsonl`
- `data/V3/processed/cluster.jsonl`

### 3.2 槽位抽取

`build_slots.py`

输入：
- `memory.jsonl`
- 邻接文本 `prev_raw` / `raw` / `next_raw`

输出：
- `data/V3/processed/memory_slots.jsonl`

固定 8 槽：
- `raw_brief`
- `event`
- `intent`
- `entities`
- `status`
- `emotion`
- `context`
- `impact`

### 3.3 hybrid 粗召回

`build_hybrid_index.py`

输入：
- `memory.jsonl`
- `memory_slots.jsonl`

输出：
- `data/V3/indexes/dense/*`
- `data/V3/indexes/bm25/*`
- `data/V3/indexes/metadata.json`

默认行为：
- dense top 50
- BM25 top 50
- union 去重

### 3.4 cluster 扩召与训练样本

`build_training_samples.py`

输入：
- `query.jsonl`
- `cluster.jsonl`
- hybrid 粗召回结果

输出：
- `data/V3/train/train_samples.jsonl`
- `data/V3/eval/eval_samples.jsonl`

规则：
- 若同 cluster 命中比例 `>= 50%` 且命中数 `>= 2`，触发整簇扩召
- 扩召后总候选上限 `150`
- 负例：`50% near-miss + 50% random_out_cluster`

### 3.5 reranker

`train_reranker.py` / `eval_offline.py`

输入：
- 动态长度训练 list
- 8 个槽位 embedding

输出：
- `data/V3/model/reranker.pt`
- `runs/v3_train/<timestamp>/...`
- `runs/v3_eval/<timestamp>/...`

模型形态：
- 继续使用 `8x8` 对齐思路
- 每个槽位一条向量
- 动态 list + padding mask

### 3.6 对 chat 导出

`export_chat_bundle.py`

输出：
- `data/V3/exports/chat_memory_bundle.jsonl`

每条至少包含：
- `memory_id`
- `cluster_id`
- `raw_text`
- `slot_texts`
- `display_text`
- `search_text`

### 3.7 对 chat 服务

`serve_retriever.py`

职责：
- 对 `_chat` 暴露 `/health`、`/retrieve`
- 返回记忆命中结果、解释字段、trace

## 4. V3 模块职责

### `data_contract/`

定义所有 JSONL 契约：
- memory
- slot
- query
- cluster
- sample
- retrieval trace

### `slots/`

负责：
- 8 槽 schema
- DeepSeek IE prompt
- JSON 校验
- warnings/evidence/confidence 规则

### `retrieval/`

负责：
- dense index
- BM25 index
- hybrid merge
- cluster expansion
- top-k trace

### `training/`

负责：
- sample collator
- reranker model
- loss
- train loop

### `evaluation/`

负责：
- Recall 系列指标
- Gain 指标
- cluster 扩召统计

### `serving/`

负责：
- 检索服务 API
- chat 导出 bundle

## 5. V3 脚本清单

- `build_dataset.py`
  - 把共享数据转成 V3 自己的数据契约
- `build_slots.py`
  - 用 IE 抽取 8 槽
- `build_queries.py`
  - 沿用当前 query augmentation 方案，输出 V3 query
- `build_hybrid_index.py`
  - 建立 dense/BM25 索引
- `build_training_samples.py`
  - 生成训练/评测动态 list
- `train_reranker.py`
  - 训练 V3 reranker
- `eval_offline.py`
  - 离线评测
- `export_chat_bundle.py`
  - 为 `_chat` 导出可直接读取的记忆包
- `serve_retriever.py`
  - 暴露检索 HTTP 服务

## 6. `_chat` 架构

`_chat` 建议目录：

```text
_chat/
  app.py
  requirements.txt
  .env.example
  configs/
  data/
    conversations/
  static/
    index.html
    app.js
    styles.css
  src/
    chat_app/
      api.py
      config.py
      models.py
      deepseek_client.py
      retriever_adapter.py
      session_store.py
      service.py
```

职责划分：

- `deepseek_client.py`
  - 调官方 chat completions
- `retriever_adapter.py`
  - 调 `_V3` 检索服务或本地导出 bundle
- `session_store.py`
  - 会话 JSONL 落盘
- `service.py`
  - 组装上下文、参考记忆、模型输入
- `api.py`
  - 提供 `/api/chat`、`/api/session/*`
- `static/*`
  - 简单聊天界面

## 7. Chat 请求链路

1. 用户在 `_chat` 输入问题
2. `_chat` 读取该 session 近期上下文
3. `_chat` 调 `_V3` 获取参考记忆
4. `_chat` 按 DeepSeek chat completions 组装 `messages`
5. `_chat` 调 DeepSeek API 生成回答
6. `_chat` 保存：
   - user message
   - assistant message
   - retrieved memories
   - prompt trace
7. 前端展示回答和本次参考记忆

## 8. DeepSeek API 参考

当前接入按官方 OpenAI-compatible 方式设计：

- Base URL：`https://api.deepseek.com`
- 使用 `chat.completions`
- 模型默认可配为 `deepseek-chat`

参考：
- https://api-docs.deepseek.com/
- https://api-docs.deepseek.com/api/create-chat-completion
