## 0. 阶段判断

当前 `_V3` 已经有：

- 数据归一化
- 槽抽取入口
- hybrid 粗召回
- `_chat` 本地聊天壳

但这还不是 `_V3/readme/问题分析.md` 所要求的“可全量测试、可看分数、可和 v2 同口径判断”的完成态。

当前缺口在主链中段：

1. 训练样本生成仍是空壳
2. reranker 训练仍是空壳
3. 离线评测仍是空壳
4. 缺少一套像 v2 那样可直接执行的 full runtime

本阶段额外新增一条优先级调整：

- `coarse rank` 的公平排序修正先登记为下一阶段事项
- 当前优先把主链补到：
  - 全量 `memory_slots.jsonl`
  - 正式 `query_slots.jsonl`
  - 更接近 v2 的 `8x8 bipartite` 神经精排器

所以这一阶段的目标不是继续做 demo，而是把 `_V3` 推到“能系统训练、系统评测、系统出分”的状态。

## 1. 阶段目标

本阶段完成后，V3 至少要满足下面 4 条：

1. 能从 `data/Processed/...` 生成 `data/V3/...` 全套主链产物
2. 能基于 `hybrid + cluster 扩召 + 8 槽特征` 生成训练样本
3. 能训练一个真正可加载的 V3 reranker，并用于在线检索
4. 能输出和 v2 风格接近的离线指标，明确区分：
   - 粗召回是否命中
   - reranker 是否带来增益

## 2. 本阶段冻结方案

### 2.1 主链

冻结为：

1. `build_dataset`
2. `build_slots`
3. `build_hybrid_index`
4. `build_training_samples`
5. `train_reranker`
6. `eval_offline`

### 2.2 reranker 形态

这一阶段先不做重型 cross-encoder，也不做边 decode 边打分。

先采用：

- 离线构建 coarse candidate
- 离线抽取 8 槽
- 离线生成候选级特征
- 小型 torch listwise reranker

这样做的目的：

- 保持和 v2 一样的“离线矩阵化、在线轻打分”方向
- 先把训练闭环、评测闭环、服务闭环打通
- 后续若要把 reranker 升级到更强模型，不需要推翻运行框架

### 2.3 训练样本策略

冻结为：

- 正例：query 标注 positives 全保留
- 负例：
  - `50% near-miss`
  - `50% random out-of-cluster`
- list 长度：动态，随正例规模变化
- cluster 扩召规则沿用：
  - `matched_count >= 2`
  - `matched_count / cluster_size >= 0.5`
  - `expanded_top_n_cap = 150`

### 2.4 评测口径

本阶段必须输出：

- `RetrievalRecall@N`
- `CoarseRankRecall@k`
- `CoarseRankRecall@P`
- `Recall@k`
- `Recall@P`
- `Gain@k`
- `MRR`
- `Top1`

## 3. 实施任务

### P3-A 特征与样本

产物：

- `agentmemory_v3/retrieval/feature_builder.py`
- `agentmemory_v3/training/common.py`
- `build_training_samples.py`

验收：

- 单条 query 可落出完整训练 list
- 样本中同时包含：
  - candidate ids
  - labels
  - candidate features
  - coarse trace
  - split

### P3-B reranker 训练

产物：

- `agentmemory_v3/models/reranker.py`
- `agentmemory_v3/models/slot_bipartite.py`
- `agentmemory_v3/training/trainer.py`
- `train_reranker.py`

验收：

- 能读取样本并训练
- 能输出权重到 `data/V3/models/slot_bipartite_reranker.pt`
- 在线检索能自动加载新权重

### P3-C 离线评测

产物：

- `agentmemory_v3/evaluation/offline_eval.py`
- `eval_offline.py`

验收：

- 能按 split 跑评测
- 能明确输出 coarse 与 rerank 两套指标
- 能写入 `_V3/runs/eval_offline/...`

### P3-D full runtime

产物：

- `runtime_full_pipeline.py`
- `doctor_data_chain.py`

验收：

- 用户可以直接按顺序运行：
  - doctor
  - prepare
  - train
  - eval

## 4. 阶段内不做

- 不在这一阶段切 Java/Go
- 不做在线学习
- 不做 LoRA 反馈闭环
- 不做复杂多模型蒸馏
- 不做 UI 美化优先级工作
- `coarse rank` 公平排序修正登记到下一阶段单独处理

## 5. 阶段完成标准

当下面 6 条全部满足，才算这一阶段完成：

1. `_V3` 不再只是 demo，可独立训练和评测
2. `_chat` 默认能加载训练后的 V3 reranker
3. 用户能按脚本跑出全量或准全量指标
4. 评测结果能区分“粗召回问题”和“精排问题”
5. 文档中明确写出 `cmd` 运行方式
6. 代码和产物边界仍保持 `_V3` 隔离
