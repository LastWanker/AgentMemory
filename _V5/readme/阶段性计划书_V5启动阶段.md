# V5 阶段性计划书_启动阶段

## 1. 目标

V5 当前阶段的主目标是：

1. 保留 `_V4` 的强 coarse
2. 研究新的联想检索能力
3. 为求职 demo 提供一个比“轮子串联”更像样的新卖点

## 2. 冻结前提

先冻结以下事实，不再反复争论：

1. `_V4` 的 coarse 已经很强
2. V5 当前源码与数据必须保持 coarse-only
3. 个人记忆任务更需要稳定召回和可解释的工程链路
4. V5 要找的是“coarse 没擅长解决”的问题

## 3. V5 的建议主线

### P0. 工程隔离完成

- 默认路径全部切到 `_V5` / `data/V5`
- `data/V5/VectorCacheV5` 作为正式缓存目录
- `_V5/chat/data/conversations` 保持空目录或只保留当前联调数据
- `_V5/runs` 保持干净

### P1. 保留强 coarse 作为精确通路

继续保留并可直接复用：

- `E5-small`
- `BM25`
- `cluster 连带`

这条路解决：

- 主体明确
- 关键词明确
- 精确找回

### P2. 新增联想通路原型

方向：

- 标签库
- 标签树
- 记忆到标签的多标签挂接
- query 到标签库的快速匹配

目标不是严格的“这个 query 的唯一正例是谁”，而是：

- 这个 query 会联想到哪些主题、脉络、情绪、对象

### P3. 先做非 LLM 在线版本

在线时尽量不再依赖 LLM API。

优先路线：

1. 离线构建标签库
2. 离线给 memory 打标签
3. 在线 query 只做 embedding 匹配
4. 由标签层级和记忆图做联想扩展

### P4. 重新定义 demo 讲法

求职 demo 口径改成：

- `Precise path`: 强 coarse，负责精确回忆
- `Associative path`: 新通路，负责联想回忆

## 4. 本阶段暂时不做

1. 不重新引入 slot/training/model 目录
2. 不把额外的大体积历史资产重新搬回 `data/V5`
3. 不急着做新的复杂 UI
4. 不让联想原型破坏现有 coarse 主链

## 5. 第一批建议产物

1. `readme/联想检索方案草案.md`
2. `scripts/build_label_inventory.py`
3. `scripts/build_memory_label_graph.py`
4. `scripts/runtime_associate_demo.py`
5. `src/.../association/` 新模块

## 6. 成功标准

V5 启动阶段的成功是：

1. 有一条清晰的新技术主线
2. 能解释它为什么是 coarse 之外的新价值
3. 有最小原型可看
4. 下个窗口能无歧义接手
