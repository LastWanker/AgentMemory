# OREO 阈值快扫与可控边界（2026-03-06）

## 1. 快速版专用阈值重搜

- 命令口径：`random=300`，`20` 个 `(threshold_coarse, threshold_association)` 组合
- 产物：`data/V5/suppressor_oreo_r2_eval/two_cohort_threshold_rescan_fast_300_20combo.json`
- 固定参数：
  - `alpha=0.25`
  - `min_delta_coarse=min_delta_association=0.0`
  - `min_margin=0.0`
  - `min_zscore_coarse=min_zscore_association=-0.5`
  - `max_drop_per_lane=1`
  - `keep_top_per_lane=3`
  - `SUPPRESSOR_USE_CALIBRATION=1`

## 2. 核心结果

20 个阈值组合结果完全一致（没有形成可调曲线）：

- `feedback_pair_removed_rate = 12.89%` (`25/194`)
- `random_dual_top1_changed_rate = 46.0%`
- `random_dual_top5_set_changed_rate = 76.67%`
- `random_dual_top5_order_changed_rate = 87.0%`

含义：

1. 当前在该配置下，阈值已不再是主要控制杆（进入“饱和触发”区间）。
2. 调阈值无法解决“feedback 提升 vs random 扰动”冲突。

## 3. 可控边界（硬上限）

基于当前数据和保险丝参数估算：

1. 反馈命中覆盖：`hit_off = 194/203`
2. `max_drop_per_lane=1` 下，按 `(query,lane)` 计的理论上限：
   - `hit_group_count = 111`
   - 上限约 `111/194 = 57.22%`
3. 再叠加 `keep_top_per_lane=3`（top3 保护）：
   - `protected_top3_hit_count = 127`
   - 可移除上限 `67/194 = 34.54%`

结论：

- 只要 `keep_top_per_lane=3` 和 `max_drop_per_lane=1` 不变，`feedback_removed_rate` 天花板本身就被压低。
- 所以“高 feedback 命中”不可能只靠阈值搜索获得。

## 4. 哪些超出当前掌控（主要矛盾）

1. **指标上限受保险丝约束**（不是模型可学习部分）。
2. **训练目标与线上决策错位**：
   - 训练按 pair/group 学分数
   - 线上按 lane 只允许极少 drop
3. **样本语义冲突和噪声**：
   - `unrelated` / `toforget` 仍有局部冲突 key
4. **分数可分性不足**：
   - 阈值区间表现出“要么全不动、要么全乱动”的两段式。

## 5. 关于“是否需要复杂门控”

你的目标方向是对的：理想状态应是“模型本体足够准，门控尽量简单”。  
当前现实是模型尚未提供稳定可分性，所以必须依赖门控兜底，导致两难明显。

## 6. 是否考虑纯记忆网络

可考虑，但不是立刻替代：

1. 纯记忆网络会更擅长记住特例；
2. 但若缺乏稳定读写规则和冲突仲裁，随机扰动可能更大；
3. 更稳妥路径是先做“检索式记忆补偿层”（可解释、可控）：
   - 对反馈表做显式 key 命中与可追踪压制
   - 再让可学习模型处理泛化部分

## 7. 参数量是否足够

当前 `oreo_type_heads_mlp_v1` 约 `553,602` 参数（`input_dim=775, hidden=(384,128), K=32`）。  
从量级看，不像“参数太小”导致完全失效；更像是数据/目标/线上约束的组合问题。
