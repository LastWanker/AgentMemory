## 3. 关键结果（冒烟）

1. 产物 A（`suppressor_oreo_r2_typeheads_artifact`）  
   - 最好组合（8 combo 冒烟）：  
     - `feedback_pair_removed_rate = 5.15%`
     - `random_dual_top1_changed_rate = 8.0%`
     - `random_dual_top5_set_changed_rate = 23.5%`
   - 观察：比“完全不触发”强，但仍未达到目标区间。

2. 产物 B（`suppressor_oreo_r2_typeheads_artifact_v2`）  
   - 最好组合（8 combo 冒烟）：  
     - `feedback_pair_removed_rate = 1.55%`
     - `random_dual_top1_changed_rate = 0%`
     - `random_dual_top5_set_changed_rate = 13.0%`
   - 观察：更稳，但反馈命中进一步下降。

3. 输出分布诊断  
   - 产物 A：`type_gap_p50 ≈ 0.004`，若 `min_type_gap=0.02` 会大量触发冲突 hold，导致几乎 no-op。  
   - 产物 B：`type_gap_p50 ≈ 0.0286`，冲突 hold 压力下降，但 `feedback` 命中仍低。

4. B3 标定结果（产物 A）  
   - 校准集 NLL：`0.6349 -> 0.2633`
   - 标定参数已限制在合理范围（温度/偏置有边界，不再出现极端值）
   - 但若沿用旧阈值（如 `0.4`）会出现“几乎不触发”，说明标定后必须重做阈值搜索。

5. B3 标定后小网格（低阈值）  
   - best（12 combo 冒烟）：  
     - `feedback_pair_removed_rate = 1.03%`
     - `random_dual_top1_changed_rate = 0%`
     - `random_dual_top5_set_changed_rate = 12.0%`
   - 观察：随机稳定性进一步增强，但 feedback 命中仍偏低。

6. B3 热开关验证（同一参数单组合）  
   - `SUPPRESSOR_USE_CALIBRATION=1`：`feedback_removed=0%`，`random_top1/top5_set=0/0%`  
   - `SUPPRESSOR_USE_CALIBRATION=0`：`feedback_removed=5.15%`，`random_top1/top5_set=9/22%`  
   - 结论：标定热开关生效，且会显著改变触发尺度。

## 4. 结论

1. 阶段B的“类型分头”已经实装并接入训练/推理/评测主链。  
2. 阶段C新增冲突指标已可用于网格评测。  
3. B3 的“标定能力”已经具备，但“标定后阈值区间”仍需系统重搜。  
4. 当前主要瓶颈仍是可分性不足：提升一点 feedback 命中就容易抬高 random 扰动。  
5. 现阶段还没有达到 `feedback 高命中 + random 低扰动` 的目标区间。

## 5. 下一步改良（建议）

1. 继续在 B3 上做“分 lane 阈值搜索自动化”（以 two_cohort 目标函数选阈值）。  
2. 在 type-head 训练时做 hard-negative 重采样，重点提升 feedback 对应候选的相对排序。  
3. 试 `lane/type` 独立 `min_delta` 与 `alpha`，避免 association 被 coarse 拖拽。  
4. 用 200 随机样本先找稳定区，再放大到 2000 全量搜索。
