给 Codex 的计划表（一步到位版）

目标
新增一个“联想通道”，与现有粗召回通道并列工作：

粗召回：负责精确记忆（命中就很准）

联想通道：负责主题/概念联想（跨事件、跨措辞，但仍然合理）
两条通道产出各自的候选列表。默认不做融合打分、不做 reranker。最终给上层 LLM 的输入可以是：粗召回 topK（精确） + 联想 topM（新颖），并且把粗召回 topK 从联想结果里剔除以避免重复。

核心架构（你最终要实现的东西）

离线构建“概念层级图”（层级中心，不是词典树）

L1：具体概念层（你要的“微积分应用题”这层），规模 ~20k 可接受

L2：分类层，规模 ~1k–3k

L3：领域/话题层，规模 <=500（理想 200）

L0：不再作为系统核心；你甚至可以完全不保留“10 个脏词”的规则。L0 仅作为抽取/中间材料，不进入联想检索主链路。

在线联想检索 = “种子概念命中 + 向上/向下传播（TTL） + 从挂载的记忆中按亮度选 topM”

必须强调的设计取舍（写进 Codex 任务书）

不训练 reranker，不做 8x8 矩阵训练，不做端到端排序学习

不依赖粗召回的相似度来给联想结果“严出”。联想通道必须能独立排序

不搞“疑似同义软连接”。同义合并阈值定死 0.85（归一化 cosine）

允许路径缺层（不是每层必须有），但建议最终都能落到 L2/L3（为空也行）

传播只走父子方向（兄弟不建边），兄弟通过“共享父节点”隐式得到

回头路不走：传播过程严格分两段：先向上，再可选向下；不允许上上下下乱跳

一、数据结构（越简单越好）

Memory 表（你已有）

memory_id

text（原文）

meta（时间、来源、对话 id…可选）

Concept 节点表（统一结构，靠 level 区分）

node_id

level in {L1, L2, L3}

name（字符串；L1 可以是“微积分应用题/某游戏/某政治事件观点”，L3 是“国际政治/学习/饮食”之类）

vec（E5-small embedding，384 维）

aliases（别名列表，可选但建议加）

边表（严格父子，保证 DAG）

child_id

parent_id

conf（0~1，来自聚类/相似度；就是你想要的“置信值”）

挂载表（记忆挂载到 L1，强烈建议）

memory_id

l1_node_id
（可选带 weight，但你倾向不用软挂载，那就不带或恒为 1）

倒排索引（用于快速从概念取回记忆）

l1_node_id -> [memory_id...]

二、离线构建（一步完成规则、一次跑通）

输入

记忆集合 M（3000+条）

E5-small 模型（384维）

输出

L1/L2/L3 节点集合 + 向量

L1->L2->L3 父子边 + conf

memory->L1 挂载 + 倒排

步骤 0：文本预处理（轻量）

可选：去掉极端长日志/代码块的噪声（不必很复杂）

不需要分段，不需要 10 个脏词上限规则（按你最新决策）

步骤 1：为每条记忆生成 L1 概念（离线 LLM，一步到位）
对每条 memory_text，调用 LLM 输出 1~5 个 L1 概念短语（中文为主），强调“概念提纯”：

允许把符号/乱句归纳成“微积分应用题/线代推导/机器学习消融实验/国际政治战争讨论”等

不要求固定层数，但 L1 必须给出

要求输出严格 JSON，便于 parse（Codex 直接按 schema 做）

同义合并（必要但极简）

将每个新 L1 概念 embed 成 vec

在现有 L1 词库中找最近邻 cosine

若 max_sim >= 0.85：合并为已有 canonical（加入 alias）

否则新增 L1 节点
最终得到 L1 节点集与每条记忆的 L1 列表（1~5）

挂载

记忆挂载到它的 L1 概念节点（可多挂）

建 l1->memory 倒排

备注：这一步是你非常认可的“记忆指向 L1 更好”。它让 L0 彻底退场。

步骤 2：生成 L2/L3 候选（仍离线 LLM，但只做“归并建议”，不建父子边）
对每个 L1 概念，让 LLM 输出：

L2 分类（1~3，可空）

L3 领域（1~3，可空，且禁止上升到“宇宙顶层”）
同义合并规则同上（0.85 合并，否则新增）。
得到 L2/L3 节点集合，并记录每个 L1 的“候选 L2/L3 名称集合”（只是候选，不直接当边）。

步骤 3：聚类从零造桥（父子边完全由聚类决定，不用 LLM 路径）
你想贯彻到底：删除 LLM 给的上溯边，只保留节点本身。那就按“中心映射”做父子：

核心思想：每个子节点指向一个父中心（唯一父），conf 由相似度给出。兄弟关系通过同父隐式获得。

具体做法（两种实现，Codex 选其一即可）：

A) 目标中心法（你给的 N/W 逻辑，推荐）

先确定父层“凝结核/中心列表”N：

L3：直接把所有 L3 节点当中心（<=500，最好 200）

L2：把所有 L2 节点当中心（1k~3k）

L1：把所有 L1 节点当子（~20k）

建边：

L2 -> L3：对每个 L2，找最相似的 L3 中心，建立边 child=L2 parent=L3，conf=cos

L1 -> L2：对每个 L1，找最相似的 L2 中心，建立边 child=L1 parent=L2，conf=cos
这种实现非常直、非常可控，且不需要 KMeans。你的“一个都不能掉队”天然满足：每个子都能找到最近的父。

B) KMeans 层级聚类法（可选）
如果你担心 L2/L3 词库里也有噪声，可以对 L2/L3 先做聚类压缩成固定 K 的中心，再做 A) 的映射。但你已经给了 L2/L3 的规模上限，其实 A) 足够先跑起来。

conf 归一化

你说 cosine 可以归一化：直接把 cos [-1,1] 映射到 [0,1] 或直接截断负值

建议：conf = max(cos, 0)（足够简单）

边的清理（极简护栏）

每个 child 只保留一个 parent（保证树状层级，避免回头路）

不做“疑似同义”边

不做兄弟边

三、在线联想检索（独立排序，不靠粗召回）

输入

query 文本

输出

联想候选 memories（按亮度排序的 topM，比如 50~200）

步骤 1：找到 L1 种子（注入点）
由于你不想让 L0 参与主链路，那注入点就直接落在 L1：

query 用 E5 编码成 q_vec

在 L1 节点向量库里做 ANN/暴力 topR（R=20~50），得到一组 seed L1 节点及其 seed_score（可以用 cos 或归一化后值）

步骤 2：传播点亮（TTL + conf + 上下代价不同）
你已经认可 TTL 点亮，那就用最干净的公式：

初始化：brightness(L1_seed) = seed_score
向上（建议先做 2 跳）：

L2 += L1 * conf(L1->L2) * α_up

L3 += L2 * conf(L2->L3) * α_up
向下（可选 1 跳，用于“把高层相关性投回更细概念”）：

L2 += L3 * conf(L2->L3) * α_down

L1 += L2 * conf(L1->L2) * α_down

建议默认参数（先跑通再调）：

α_up = 0.8

α_down = 0.4

TTL_up = 2（L1->L2->L3）

TTL_down = 1（L3->L2 或 L2->L1 任选）

限制扩散（防止全亮，但不复杂）：

每个节点传播时只保留 topB 的下游/上游（B=200 对 L1 足够；对 L2/L3 可更小）

步骤 3：从亮 L1 直接取回记忆（联想通道独立“选条”）

对所有 L1 节点按 brightness 排序，取 topT（例如 200~1000）

用倒排 l1->memory 拉候选集

记忆亮度定义（不看文本相似，只看结构点亮）：

brightness(memory) = max_{l1 in attached(memory)} brightness(l1)

或 top2 平均，防止单点误触发（可选）

输出 topM（比如 100）。
可选去重：同一对话/同一片段只保留最亮的几条，保证结果“像联想而不是刷屏”。

步骤 4：与粗召回并列，不做复杂融合

粗召回通道照旧输出 topK_precise

联想通道输出 topM_assoc

将粗召回 topK_precise 的 memory_id 从 topM_assoc 中剔除（你明确提出的规则）

上层 LLM 输入可以直接：precise + assoc（两段分开展示，避免误解）

四、关于你问的“E5small够不够？要不要词向量？”
结论：E5-small 完全够用，而且你已经在用它，统一一个模型最省心。
你写的“凝结核列表N vs 待分类词表W”的余弦匹配逻辑，本质就是“向量最近邻映射”，非常适合你的“一个都不能掉队、每个子找一个父”的建边方式。词向量（word2vec/fastText）在中文短词上可能会带来额外噪声与维护成本，不建议在第一版引入。
