我们不再指望“句向量粗召回+reranker”去硬把联想学出来，而是专门搭一条“概念化—分层—传播”的并行检索通道，让它像人的联想一样先找方向、再沿着方向把相关记忆点亮出来，最后直接按“被点亮的强度”把记忆挑出来；精确回忆仍然交给你原来的粗召回通道，两者并行，各司其职，互不拖累。

具体怎么做呢？离线阶段，我们把每条记忆先“提纯成概念”。你之前担心脏文本、数学符号、过细的应用题这种东西会让系统失控，所以我们不再执着于从原文里抠十个关键词当作底层标签，而是把整条记忆文本交给 LLM，让它输出 1~5 个“L1 概念”，这些概念是你真正希望被联想到的东西，比如“微积分应用题”“国际政治战争讨论”“对某社交软件的吐槽”“某款游戏的沉迷与倦怠”。这一步的作用是把杂乱无章的自然语言压缩成一组可检索、可泛化、可复用的概念点，并且每条记忆都直接挂到这些 L1 概念上——这相当于你为联想通道准备了一张“概念倒排表”，以后点亮了某个概念，就能立刻捞到挂在它下面的记忆。

接着我们再往上做两层更粗的抽象：L2 分类和 L3 领域/话题。它们不是为了追求哲学上完美的分类学，更像是给联想提供“更高层的方向”。LLM 可以给每个 L1 概念建议 0~3 个 L2/L3，但这里我们只把这些名字当作“节点来源”，不让 LLM 决定谁是谁的父亲，因为你想贯彻“联想能力要由相似与聚类来塑形”。于是父子关系全部由向量相似来从零造桥：我们用 E5-small 给 L1/L2/L3 的节点都编码成向量，然后把每个 L1 指向它最相似的 L2（唯一父亲，避免回头路）【这里其实有点问题，因为我们其实希望如何利用E5的特性让同义词区分开呢？这样就不必严格一个父亲了。】，每个 L2 指向它最相似的 L3，同样唯一父亲，并且把这个余弦相似度直接当作边的置信值 conf。这样一来，层级结构就不是“词典式的树”，而是“语义中心的层级映射”：一个概念被哪个分类吸收、哪个分类被哪个领域吸收，全靠语义相近度决定，而且一个都不会掉队，因为每个节点都能找到最近的父节点。

在线阶段，联想检索的入口也不再是 L0 关键词，而是直接从 L1 概念层注入。用户输入 query 之后，我们用同一个 E5-small 把 query 编成向量，在 L1 概念库里做最近邻搜索，取一批最相近的 L1 概念作为种子节点，每个种子节点带一个初始亮度（可以就是相似度）。然后开始你认可的“TTL 点亮传播”：亮度沿着父子边向上走两三跳，把更抽象的 L2/L3 节点点亮；必要时再做一跳向下，把高层的相关性回灌到更具体的 L1 概念。传播过程中亮度会衰减，并且同时乘上边的置信值 conf，这样传播既反映“走一层要付出代价”，也反映“这条父子关系到底靠谱不靠谱”。你可以把它理解为：越接近 query 的概念节点越亮，越远的节点亮度自然变暗，但如果某条抽象链路非常可信，它仍然能把“同主题不同事件”的兄弟概念点亮起来。

关键点来了：联想通道的最终排序不再依赖原来的粗召回相似度，而是完全由“亮度”决定。我们取亮度最高的一批 L1 概念，从倒排表里把挂在这些概念下面的记忆全部捞出来，然后给每条记忆一个记忆亮度（例如它所挂载的 L1 概念亮度的最大值或 top2 平均），按这个亮度直接排序输出 topM。因为这套评分来自图传播而非文本相似，所以它天然更擅长把“词面不重合但主题同构”的东西拉出来：用户谈巴以，种子概念点亮“战争/国际冲突/国际政治”，通过上层领域再回灌，就能把“俄乌相关的概念节点”也点亮，从而把俄乌相关记忆捞进候选；用户说电子阳痿没劲，概念层命中“兴趣衰退/游戏倦怠”，点亮后自然能把过去的游戏记忆带出来，即使没有出现具体游戏名。

最后，这条联想通道与原有粗召回通道是并列的。粗召回继续负责“你问啥我就把那条精准记忆翻出来”，联想通道负责“你提到这个方向，我再想起一些同类/同构/相关的东西”。两条通道默认不做复杂融合、不训练 reranker、不做统一损失；唯一的协调规则就是：粗召回的 topK 结果从联想结果里剔除掉，避免重复刷屏。上层聊天模型看到的则是两段清晰分区的证据：一段是精确回忆，一段是联想补充。这样体验上很像人：我先准确回答你问的那件事，同时我也顺带想起一些可能有用的相关经历，但我不会把“联想”伪装成“精确事实”。
