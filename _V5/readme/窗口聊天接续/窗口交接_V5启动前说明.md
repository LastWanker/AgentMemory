# V5 窗口交接_V5启动前说明

## 1. 当前定义

V5 现在是一条干净的 coarse-only 主链。

现行检索路径固定为：

1. `multilingual-e5-small` dense recall
2. `BM25` lexical recall
3. `cluster` 连带扩召
4. `_V5/chat` coarse-only 展示

这条链路已经可以独立准备数据、构建索引、离线评测、服务化和聊天接入。

## 2. 当前边界

V5 目录里只保留现行主链需要的东西：

- `processed/{memory,query,cluster}.jsonl`
- `VectorCacheV5/users/{manifest,memory_ids,query_ids,memory_coarse,query_coarse}`
- `indexes/{dense_artifact,dense_matrix,bm25}`
- `exports/chat_memory_bundle.jsonl`

源码侧只保留：

- `data_contract`
- `encoder`
- `evaluation`
- `query_utils`
- `retrieval`
- `serving`
- `utils`

不再保留 slot、training、model 目录。

## 3. 现阶段要做什么

V5 接下来要在这条稳定主链之上继续做两类工作：

1. 把 coarse 基线打磨成可靠工程底座
2. 在 coarse 之外新增联想检索/关联回忆通路

第二条才是后续新增价值的主要方向，但必须建立在第一条始终可运行的前提上。

## 4. 已确认的工程习惯

### 4.1 命令

统一用：

```bat
cmd /c "..."
```

不要在 PowerShell 外层直接写复杂重定向或管道。

### 4.2 看文件

- 搜索优先 `rg`
- 不要一次性读取大文件
- 需要局部定位时优先 `findstr /n`

### 4.3 运行环境

- 默认用 `.venv\Scripts\python.exe`
- E5 默认 `local_files_only: true`
- E5 默认 `offline: true`

## 5. 当前可直接使用的路径

- 配置：`_V5/configs/default.yaml`
- 数据根：`data/V5/`
- 缓存根：`data/V5/VectorCacheV5/`
- 聊天器：`_V5/chat/`
- 运行输出：`_V5/runs/`

## 6. 推荐验证命令


## 7. 下一步方向

下一阶段新增能力时，优先考虑：

1. 标签库/标签树
2. 记忆到标签的离线挂接
3. query 到标签空间的快速匹配
4. 基于标签或图的联想扩散
