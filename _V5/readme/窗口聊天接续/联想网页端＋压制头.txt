每个槽位都对应一个 key 向量和一个 value 向量，匹配时用 dot 或 cosine 算相似度，
再用一个很尖的注意力（比如温度很小的 softmax）得到对各槽位的权重，

于是模型会自然形成一种“局部记忆”：某些特例组合会被同一个槽位负责，
邻域里相似的 query 会激活同一个槽位从而产生一点点泛化，但不相干的 query 基本不会激活那个槽位，
