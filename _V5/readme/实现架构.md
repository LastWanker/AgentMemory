# V5 实现架构

更新时间：2026-03-02

## 1. 目标

V5 是一个纯 coarse-only 的个人记忆检索工作区。

当前主链只做四件事：

1. 生成 V5 自己的数据契约
2. 构建 `multilingual-e5-small` 粗召回缓存
3. 构建 dense + BM25 + cluster 的混合检索
4. 给 `_V5/chat` 提供本地或 HTTP 检索能力

## 4. 主链

### 4.1 数据准备

`build_dataset.py`

输出：

- `memory.jsonl`
- `query.jsonl`
- `cluster.jsonl`

### 4.2 E5 缓存

`build_e5_cache.py`

行为：

- memory 侧直接编码 `raw_text`
- query 侧编码 `rewrite/query/raw_query` 的有效文本
- 模型固定走 `multilingual-e5-small`

### 4.3 混合检索

`build_hybrid_index.py`

行为：

- dense 检索
- BM25 检索
- union 去重
- cluster 连带扩召

### 4.4 chat 导出

`export_chat_bundle.py`

导出字段：

- `memory_id`
- `cluster_id`
- `raw_text`
- `display_text`
- `search_text`

### 4.5 服务与聊天

`serve_retriever.py` 提供 `/health` 和 `/retrieve`。

`_V5/chat` 默认优先：

1. 本地 V5 retriever
2. HTTP retriever
3. bundle fallback

无论哪条路，展示逻辑都只按 coarse 结果工作。
