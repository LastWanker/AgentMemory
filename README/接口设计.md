### 1.3 `eval_*.jsonl` / `eval_*_followup.jsonl`
- eval 行（`to_eval_rows`）：
```json
{"query_id":"...", "query_text":"...", "expected_mem_ids":["m1","m2"], "meta":{}}
```
- followup 行（`to_followup_rows`）：
```json
{"query_id":"...", "query_text":"...", "positives":["m1","m2"], "meta":{}}
```

说明：
- 训练与评测都兼容读取 `positives` 或 `expected_mem_ids`。
- 旧字段 `candidates/hard_negatives` 仅兼容检测，不参与候选构造。

## 2) 运行时衍生契约（容易忽略）

### 2.1 `memory_jsonl -> MemoryItem` 会发生切块
来源：`src/memory_indexer/ingest.py::build_memory_items`

- 默认 `chunk_strategy=sentence_window`，`max_sentences=3`
- 单块保留原 `mem_id`
- 多块会变成 `mem_id#c1`, `mem_id#c2`, ...

### 2.2 正例匹配支持 chunk 后缀
来源：`scripts/memory_indexer/train_reranker.py::is_positive`

- 若期望正例为 `c000123`，则 `c000123#c1` 也视为正例
- 避免切块后训练样本被误判为无正例

## 3) Python 核心接口

### 3.1 模型对象（`src/memory_indexer/models.py`）
- `MemoryItem`: `mem_id, text, created_at, source, meta, summary, keywords`
- `EmbeddingRecord`: `emb_id, mem_id, encoder_id, strategy, dims, n_vecs, vecs, coarse_vec, aux`
- `Query`: `query_id, text, context, encoder_id, strategy, q_vecs, coarse_vec, aux`
- `RetrieveResult`: `mem_id, score, coarse_score, debug, features, route_output`
- `RouteOutput`: `policy, weights, selected_ids, metrics, explain, scores`

### 3.2 编码器接口（`src/memory_indexer/encoder/base.py`）
```python
encode_tokens(text) -> (token_vecs, token_strings)
encode_query_sentence(text) -> vec
encode_passage_sentence(text) -> vec
```

当前实现：
- `SimpleHashEncoder`
- `HFSentenceEncoder`
- `E5TokenEncoder`

### 3.3 向量化、索引、检索
- `Vectorizer.make_group(token_vecs, tokens, text)`
  - 当前策略：`token_pool_topk`、`cluster_centers_r`
- `CoarseIndex.add/search`
- `LexicalIndex.add/search`
- `MemoryStore.add(item, emb, tokens)`
- `Retriever.retrieve(q, top_n, top_k, candidate_mode)`
- `Router.route(scored_features)`（`soft / half_hard / hard`）

### 3.4 Pipeline 入口（`src/memory_indexer/pipeline.py`）
- `build_memory_index(...)`
- `retrieve_top_k(...)`

## 4) 脚本接口（CLI）

### 4.1 聊天处理侧
- `scripts.memory_processer.build_chat_memory`
  - 关键参数：`--config --input --session-id --segmentation-mode --fixed-sim-threshold --novelty-threshold --min-segment-len --no-cross-session-merge --global-merge-sim-threshold`
  - 输出：`raw/dedup/memory` 三份 JSONL
- `scripts.memory_processer.build_chat_queries`
  - 关键参数：`--memory-in --supplemental-queries --eval-out --followup-out`
  - 输出：`eval_out + followup_out`
- `scripts.memory_processer.build_chat_supplemental_queries`
  - 关键参数：`--memory-in --out --base-url --model --api-key-env --api-key-file --concurrency --max-clusters --max-per-layer --fallback-only`
  - 输出：supplemental JSONL

### 4.2 数据集合并侧
- `scripts.memory_indexer.build_merged_dataset`
  - 输入：`memory-a/eval-a + memory-b/eval-b + optional eval-c`
  - 输出：`memory_<dataset>.jsonl`、`eval_<dataset>.jsonl`
  - 行为：重复 `mem_id` 去重；冲突 `query_id` 自动重命名；校验 positives 是否能在 merged memory 中找到

### 4.3 评测侧
- `scripts.memory_indexer.eval_router`
  - 关键参数：
    - 数据与采样：`--dataset --max-eval-queries --eval-sample-mode --eval-sample-seed`
    - 路由与 ablation：`--policies --ablation --ablation-groups --consistency-pass`
    - 检索：`--top-n --top-k --candidate-mode`
    - scorer：`--use-learned-scorer --reranker-path --scorer-batch-size`
    - 缓存：`--cache-alias --rebuild-cache --no-cache-signature`
    - 并行：`--eval-workers --torch-num-threads`
    - 产物：`--run-dir`
  - 产物：`eval.log.txt`、`eval.metrics.json`、`config.snapshot.json`

### 4.4 训练侧
- `scripts.memory_indexer.train_reranker`
  - 关键参数：
    - `--loss-type pairwise|listwise`
    - `--neg-strategy random|ranked`
    - `--neg-per-pos --list-size --max-queries`
    - `--use-cardinality-head --cardinality-loss-weight --cardinality-k-max`
    - `--save-path --export-weights-to --run-dir`
  - 产物：权重 `.pt`、`train.log.txt`、`train.metrics.json`、`config.snapshot.json`
- wrappers：
  - `train_pairwise_reranker`（强制 `loss_type=pairwise, neg_strategy=ranked`）
  - `train_listwise_reranker`（强制 `loss_type=listwise, neg_strategy=ranked`）

## 5) Run 与缓存契约

### 5.1 `run_dir` 约定
- 训练：`train.log.txt`、`train.metrics.json`、`config.snapshot.json`
- 评测：`eval.log.txt`、`eval.metrics.json`、`config.snapshot.json`

### 5.2 registry
- 文件：`runs/registry.csv`
- 更新函数：`scripts/memory_indexer/runtime_utils.py::upsert_run_registry`

### 5.3 缓存
- 目录：`data/VectorCache/`
- query 缓存：`eval_cache_<signature|alias>.jsonl`
- memory 缓存：`memory_cache_<signature|alias>.jsonl`
- `eval_router` 中：
  - query 签名包含 `data_fingerprint + 采样参数`
  - memory 签名偏向“记忆侧配置”，并通过逐条 `mem_id/text/encoder/strategy/coarse_role` 校验兜底复用安全

## 6) 当前行为性约束（必须知晓）
- `eval_router` 与 `train_reranker` 会将 `candidate_mode` 收敛为 `coarse`。
- query 里的 `candidates/hard_negatives` 不进入候选池逻辑。
- `HFSentenceEncoder.encode_tokens()` 仍是 placeholder token 向量；正式 token 级实验应搭配 `E5TokenEncoder`。
