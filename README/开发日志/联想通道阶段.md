## 3. 阶段目标与冻结口径
- V5 从“仅 coarse 粗召回”升级为“双通路并列”：`coarse + association`。
- 联想通路不是 reranker，不并入 coarse 排序；最终给 LLM 两路材料并做去重。
- 在线注入主线冻结为 dense（E5），不走在线 LLM 提纯。
- 注入点 L1/L2/L3 公平竞争，不做分层配额。
- 点亮是“状态机受控传播”，不是最短路搜索。
- L1 下溯定义为桥接扩散，TTL 规则冻结：初始 2，上溯不扣，下溯/桥接扣。

## 4. 需求推进时间线（从聊天恢复到联想落地）
1. 从“上个窗口”恢复后，先确认已有图资产可用（`data/V5/association`），再进入接线阶段。
2. 新增“图查询脚本+网页展示”需求：输入 `L1/L2/L3 + 精确文本`，返回父子孙结构，不命中则明确无结果。
3. 图页面先补“桥接可视化”与后端同步返回，再围绕布局问题迭代。
4. 点亮算法讨论冻结：dense 主注入、TTL 状态机、bridge 作为 L1 下溯、回头路禁止、受控抽样下溯。
5. 写入方案文档：`联想通路全量提纲.txt` 与 `点亮算法实现草案.md`。
6. 联想召回接入 Chat 输入层：提示词明确“联想得来，不等于用户显式提到”。
7. Chat UI 改成并列展示：右侧 coarse，左侧 association（前 5），并显示高亮 tags。
8. 新增 activation 页面，展示点亮路径与节点亮度，支持独立查看。

## 5. 可视化迭代主线（图页面与激活页面）
- 第 1 轮：增加“命中节点水平线上的桥接小气泡（最多 6）”。
- 第 2 轮：修复 L1 横向漂移，要求命中节点视觉中心更稳定。
- 第 3 轮：激活页从树形过渡到同心圆布局（L3 在中圈附近，后续加外层桥接圈）。
- 第 4 轮：新增“仅桥接命中但无父亲”的外圈层，桥接线改暖白，半径压缩。
- 第 5 轮：缩放逻辑修正为同心圆间距等比例，不只缩部分层。
- 第 6 轮：重写角度/排队算法目标，主目标切换为“总连线长度最小化”。
- 第 7 轮：新增交互，点击任意气泡时，直接相关连线“满亮+2x 加粗”。

## 6. 关键踩坑与修复
- E5 与 LLM 概念混淆：早期将“卡住”归因不清，后明确是 E5 离线加载/联网行为问题，要求强离线。
- 相似度阈值误判：`aliases` 过多，怀疑相似度分布异常，补了 `check_e5_similarity.py` 验证前缀与相似度矩阵。
- 缓存策略不足：明确要求 LLM 抽取结果与向量 cache 可复用，避免重复花费和重复编码。
- 图布局回归：新增桥接元素后触发横向错乱，反复通过居中约束和布局算法重写修复。
- 前端显示异常：部分节点显示 ID 而非 tag，后续在展示层做名称兜底与映射修正。
- 视觉与交互冲突：窗口宽度变化导致排布失稳，后转向固定层级+更强布局约束。
