# 窗口交接：压制头时期

## 1. 当前状态（一句话）
V5 已进入“双通路召回 + 反馈驱动抑制”阶段：联想链路可用，反馈链路已落地，suppressor 可热开关，当前主任务是“降噪调优与指标验收”。

## 2. 已完成（接手无需重做）
- 联想通路：
  - coarse + association 并列召回，association 已接入 LLM 提示词。
  - 图页与 activation 页可用，支持点亮可视化与节点连线高亮。
- Chat 交互：
  - 支持“仅召回模式”开关。
  - 支持“记忆偏好”开关（控制 suppressor）。
  - 响应流程拆成 retrieve/respond 两段，先展示召回再补 LLM。
- 反馈采集：
  - 每张记忆卡有“无关/遗忘”按钮。
  - 事件写入 `_V5/chat/data/feedback/feedback_events.jsonl`。
  - 同 query 重复选择会去重，CandidateCache 会自动扣除已选 self。
- 抑制头：
  - 训练、评估、调参、对比脚本齐备。
  - runtime 已接入 coarse/association 两 lane 的后处理。

## 3. 关键开关与位置
- 前端开关（`_V5/chat/static/index.html`）：
  - `仅召回模式`：跳过 LLM，直接停在“记忆已召回”。
  - `记忆偏好`：请求里传 `memory_preference_enabled`。
- 后端配置（`_V5/chat/src/chat_app/config.py`）：
  - `SUPPRESSOR_ENABLED`
  - `SUPPRESSOR_ARTIFACT_DIR`
  - `SUPPRESSOR_THRESHOLD`
  - `SUPPRESSOR_ALPHA`
  - `SUPPRESSOR_MIN_DELTA`
  - `SUPPRESSOR_MIN_MARGIN`
  - `SUPPRESSOR_MIN_ZSCORE`
  - `SUPPRESSOR_MAX_DROP_PER_LANE`
  - `SUPPRESSOR_KEEP_TOP_PER_LANE`
  - `SUPPRESSOR_EXTRA_CANDIDATES`

## 4. 关键数据文件
- 反馈事件（主）：`_V5/chat/data/feedback/feedback_events.jsonl`
- 会话旧账：`_V5/chat/data/conversations/*.jsonl`
- 概念图资产：`data/V5/association/*`
- 抑制模型产物目录（默认）：`data/V5/suppressor_newfb/`（以实际训练输出为准）

## 5. 常用命令（按推荐顺序）
1. 构建训练集  
`cmd /c ".venv\\Scripts\\python.exe _V5\\scripts\\build_suppressor_dataset.py ..."`
2. 训练 suppressor  
`cmd /c ".venv\\Scripts\\python.exe _V5\\scripts\\train_suppressor.py ..."`
3. 离线评估  
`cmd /c ".venv\\Scripts\\python.exe _V5\\scripts\\eval_suppressor.py ..."`
4. 双 cohort 调参  
`cmd /c ".venv\\Scripts\\python.exe _V5\\scripts\\tune_suppressor_two_cohorts.py ..."`
5. 随机主题对比（2000 canonical_name）  
`cmd /c ".venv\\Scripts\\python.exe _V5\\scripts\\compare_memory_preference_top5.py ..."`

## 6. 当前验收口径（冻结）
- cohort A（反馈 query，约 200）：
  - 重点看“被标注 memory 是否从 top5 消失/明显后移”。
- cohort B（随机 query，约 2000）：
  - 重点看 top1/top5 扰动低。
  - 目标：`dual_changed_count` 约 10%~20% 区间，而不是高比例全变。
- 结论判定：
  - A 组有效压制 + B 组低扰动，才算调参成功。

## 7. 未完成与下一步
1. 指标升级：把“目标 memory 是否被压下去”做成主指标，不只看排序是否变化。
2. 调参降噪：优先调 threshold/alpha/min_margin/min_zscore/max_drop_per_lane。
3. 模型优化：继续优化“抚平无关组合”的能力，减少随机 query 误伤。
4. 门控策略：复杂上下文门控后置，但基础门控降噪应继续推进。

## 8. 主要风险
- 风险 1：只追求反馈集命中，会导致随机 query 全局扰动偏大。
- 风险 2：反馈样本分布偏窄，模型学到“关键词大棒”。
- 风险 3：静态资源缓存导致前端误判“改动未生效”（需注意版本号与强刷）。
- 风险 4：PowerShell 读取大文件会卡，排查时继续采用分段读取/脚本读法。

## 9. 交接备注
- 当前阶段不缺“功能入口”，缺“稳定指标闭环”。
- 建议接手后第一件事不是继续加新功能，而是跑一轮固定口径 A/B 评测并记录参数-指标曲线。
